HOW TO READ THIS DOCUMENT
────────────────────────────────────────────────────────────────────────────────

  Each phase lists BUILD STEPS in execution order.
  Each step lists: files to create/modify, what it depends on, what it unlocks.
  Arrows show the dependency chain:
      Step A ──► Step B    means "B depends on A being done"

  Phases are numbered but not all within a phase are strictly sequential —
  within a phase, the order matters. Between phases, the order inside the
  brackets shows the recommended sequence.

  [P0] ──► [P1] ──► [P2a / P2b] ──► [P3] ──► [P4] ──► [P5] ──► [P6]

  Where P2a and P2b can be done in parallel (they depend on P1 but not
  on each other).

  │        def get_usage_today(cls, user) -> int:                                │

  │        @classmethod                                                          │
  │        def _check_rate_limit(cls, user):                                     │

  │      Body: { "message": "user question" }                                   │
