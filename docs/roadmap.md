# Roadmap

## Target date

Target launch: January 1, 2027.

## Build logic

The roadmap is curriculum-first.
The product does not launch because the bot is impressive.
It launches because the learning experience is useful and coherent.

## Phase 1 — Curriculum foundation

Focus:
- finalize module structure,
- expand lesson/activity coverage,
- establish tone, voice, and templates,
- get enough content depth for a real learning product.

Success looks like:
- the curriculum architecture is stable,
- lessons follow one consistent format,
- activities build on previous work,
- contributors can write against a clear standard.

## Phase 2 — Lesson delivery system

Focus:
- content storage,
- lesson viewer,
- module navigation,
- progress tracking,
- clean reading UX.

Success looks like:
- users can move through modules cleanly,
- mark progress,
- and consume content without friction.

## Phase 3 — Contributor operations

Focus:
- onboarding docs,
- contribution workflow,
- repo clarity,
- content review standards.

Success looks like:
- new contributors can join and be productive quickly,
- the docs do not mislead them,
- and the repo has one clear source of truth.

## Phase 4 — Launch polish

Focus:
- onboarding flow,
- lesson sequencing,
- content QA,
- design cleanup,
- stable hosting and deploy flow.

Success looks like:
- the experience feels intentional, not patched together,
- and a new user can get value immediately.

## Phase 5 — Post-launch options

Possible later work:
- private bot examples turned into case studies,
- AI explanation layer,
- signal interpretation tools,
- premium tiers.

These are optional — pursued only after the core learning product is strong.

## Not launch critical

The following are explicitly not required for launch:
- public bot dashboard,
- signal explorer as a core feature,
- AI assistant,
- advanced monetization layers,
- public positioning around live trade intelligence.
